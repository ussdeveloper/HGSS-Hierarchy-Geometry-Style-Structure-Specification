# Contributing to HGSS

Thank you for your interest in contributing to HGSS! This document provides guidelines for contributors.

## Code of Conduct

This project follows a code of conduct to ensure a welcoming environment for all contributors.

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or request features
- Provide detailed descriptions and examples
- Include HGSS version and environment details

### Submitting Changes
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Make your changes
4. Add tests if applicable
5. Run the test suite: `npm test`
6. Commit your changes: `git commit -m "Add my feature"`
7. Push to your branch: `git push origin feature/my-feature`
8. Submit a Pull Request

### Development Setup

```bash
git clone https://github.com/sulacolab/hgss.git
cd hgss
npm install
npm test
```

### Coding Standards

- Use consistent indentation (2 spaces)
- Follow JavaScript/TypeScript best practices
- Write clear, concise commit messages
- Add JSDoc comments for functions
- Update documentation for API changes

### Testing

- Write unit tests for new features
- Ensure all tests pass before submitting
- Test conversions between formats
- Validate against the JSON schema

### Documentation

- Update README.md for significant changes
- Add examples for new features
- Keep API documentation current
- Use clear, simple language

## Types of Contributions

### Code
- Bug fixes
- New features
- Performance improvements
- Code refactoring

### Documentation
- Tutorials
- API documentation
- Examples
- Translations

### Testing
- Unit tests
- Integration tests
- Performance benchmarks

### Design
- UI/UX improvements
- Logo and branding
- Website design

## Review Process

1. All submissions require review
2. Reviews focus on code quality, tests, and documentation
3. Changes may be requested before merging
4. Maintainers will merge approved changes

## Recognition

Contributors will be recognized in:
- CHANGELOG.md
- GitHub contributors list
- Project documentation

## Questions?

Feel free to ask questions in GitHub Issues or Discussions.
